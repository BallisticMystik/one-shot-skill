# Final Report Template

Use this to present finished work clearly.

## Standard Format

```text
Outcome:
<what is now true>

Changes:
- <file/module>: <behavior changed>

Decisions:
- <important decision or assumption>

Verification:
- <command>: <result>

Residual risk:
- <risk or "none known">

Deferred:
- <follow-up or "none">
```

## Product Handoff Format

For user-facing product work:

```text
What shipped:
<plain-language product outcome>

How it behaves:
<before/after or workflow description>

Quality checks:
- UX:
- Responsive:
- Accessibility:
- Tests/build:

Known limits:
<honest limitations>
```

## Review Format

For review/checkpoint work, lead with findings:

```text
Findings:
- Severity: file:line — issue and correction

Open questions:
- ...

Summary:
...

Verification:
...
```

Keep the final concise. Do not bury blockers after summaries.
